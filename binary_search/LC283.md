## 283. Move Zeroes

### 一刷 (7/23/2026) ❌ 分支判断依赖了 slow 位置的中间状态，而不是 fast 读到的原始数据

我的代码：

```java
class Solution {
    public void moveZeroes(int[] nums) {
        int slow = 0;
        for (int fast = 1; fast < nums.length; fast++) {
            //         ❌③ 在这版里恰好无害（fast=0 轮 condition 恒 false），
            //            但标准写法必须从 0 起：靠 slow==fast 的自交换 no-op 推进
            if (nums[slow] == 0 && nums[fast] != 0) {
                // ❌❌① 核心错误：condition 看了 nums[slow] → slow 只能停在 0 上才干活，
                //       没人推它越过开头的非零段 → [1,0,2] 全程空转，原样输出（静默错误）
                int temp = nums[fast];   // ❌② swap 手误：temp 该存将被覆盖的 nums[slow]
                nums[slow] = nums[fast];
                nums[fast] = temp;       //    → nums[fast] 被写回自己，0 消失、非零 element 被复制
                slow++;
            }
        }
    }
}
```

**①的根因**：slow/fast 的分工是 fast 读、slow 写。`nums[0..slow)` 是已整理区，
内容由算法维护，是输出不是输入——**分支判断只能依赖 fast 读到的原始数据，
不能依赖 slow 位置的中间状态**。和 27 二刷同族：27 是 fast 在分支里没人推进，
283 是 slow 的推进被锁死在错误的 condition 里，现象都是 pointer 卡死 + 静默错误。

**②**：swap 三行的经典手误。temp 的唯一使命是保住那个即将被覆盖的值，
写完默念一遍数据流，或拿两个 element dry run 一轮。

**自测点**：不看答案说出 283 的 if condition 该看谁的值、为什么不能看 nums[slow]

- swap 的 temp 该存哪一个

### 正确写法

```java
class Solution {
    public void moveZeroes(int[] nums) {
        int slow = 0;
        for (int fast = 0; fast < nums.length; fast++) {  // 从 0 起
            if (nums[fast] != 0) {          // 只看 fast 读到的原始值
                int temp = nums[slow];      // temp 存将被覆盖的 nums[slow]
                nums[slow] = nums[fast];
                nums[fast] = temp;          // 0 被换到 fast 的位置
                slow++;
            }
        }
    }
}
```

开头是非零段时 slow == fast，swap 是自交换 no-op，两个 pointer 同步推进——
这就是 fast 必须从 0 起的原因。

### 复杂度分析

**Time: O(n)**：fast 每轮必进一格，恰好 n 轮，每轮 O(1)。
**Space: O(1)**：in-place swap，只有 slow/fast/temp。

对比 O(n) 额外数组做法（非零抄进新数组再补 0）：swap 版省掉 O(n) space，
且天然保持非零 element 的相对顺序。

### 沉淀

- **slow/fast 铁律：判断看 fast，写入用 slow**——`nums[0..slow)` 是输出区，
  拿它当分支依据 = 让算法依赖自己的中间产物，invariant 直接崩
- **swap 的 temp 存"将被覆盖的值"**：三行写完 dry run 两个 element 验一遍
- **pointer 卡死一族的信号**：输出和输入一模一样（空转）或某段被复制
  （单侧推进）→ 先检查每个分支里该动的 pointer 是否都有人推
