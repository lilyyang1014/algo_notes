### 二刷 (7/22/2026) ⚠️ 写出来了，但 return 错

**卡在哪**：代码结构能写对，但 loop 退出后说不清 left 和 right 各自指着什么，
return left 还是 right 靠猜（猜了 left，错）。

**真正的坎**：没把「loop invariant 决定退出时指针语义」和这题连起来——
其实就是 34 那套 lowerBound 思想，只是比较对象从 `nums[mid]` 换成了 `mid²`。

我的代码（错误点标在旁边）：

```java
class Solution {
    public int mySqrt(int x) {
        int left = 0, right = x;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (mid * mid == x) {
                //  ↑❌ int overflow：x 接近 MAX_VALUE 时 mid ≥ 46341，mid² 爆 int
                //     要写 (long) mid * mid，或除法写法 mid > x / mid
                return mid;
            } else if (mid * mid > x) {
                right = mid - 1;
            } else if (mid * mid < x) {
                left = mid + 1;
            }
        }
        return left;   // ❌❌ 核心错误：应该 return right（推导见下）
    }
}
```

Dry run（错误代码，x = 8，正确答案 floor(√8) = 2）:

- 初始 left=0, right=8
- 轮1: mid=4, 16 > 8 → right=3
- 轮2: mid=1, 1 < 8 → left=2
- 轮3: mid=2, 4 < 8 → left=3
- 轮4: mid=3, 9 > 8 → right=2
- left=3 > right=2 → 退出
- return left = **3 ❌**（正确是 right = 2）——和 34 的 `return mid` 一样是**静默错误**，
  不会崩，只会答案多 1

### 核心思路（一句话）

**求 floor(√x) = 最后一个「平方 ≤ x」的数。left 停在第一个 > x 的位置（等号被提前拦截，所以是 > 不是 ≥），它前一格就是答案 → return right。**

退出时 `right = left - 1`，即 **right² < x < left²**，right 就是 floor(√x)。

### 代码

```java
class Solution {
    public int mySqrt(int x) {
        int left = 0, right = x;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            long sq = (long) mid * mid;          // 防 overflow
            if (sq == x)      return mid;        // perfect square 直接接住（答案唯一，可以提前 return）
            else if (sq > x)  right = mid - 1;
            else              left = mid + 1;
        }
        return right;                            // last mid² ≤ x = floor(√x)
    }
}
```

### 本题独有：`==` 提前 return 为什么在这里是对的

34 的自检信号说「lowerBound 出现 `==` 分支基本就是写错了」，
但 69 里满足 `mid² == x` 的 mid **最多只有一个**（平方根唯一），找到即答案，
提前 return 安全——34 那条规则的前提是「答案是重复区域的边缘」，本题不满足。
（通用判定规则 → 见 Notion 二分「易错点」最后一条）

> return left / right 的通用规则和 704/35/34/69 对照表 → 见 Notion 二分「循环外 return 谁」

### 沉淀：其他

- 69 = 「binary search on answer space」入门题：搜索对象不是数组下标，而是答案本身，
  靠 `mid²` 这个 monotonic 函数判定。后续 875 / 1011 都是这个套路
- `mid * mid` 必须防 overflow：`(long) mid * mid` 或 `mid > x / mid`。面试常追问
- x = 0：left=right=0，mid=0，`0 ≤ 0` / `0 == 0` 均安全，不用特判
- Time: O(log x) / Space: O(1)

### Interview pitch (练口述)

> "This is binary search on the answer space — I search integers in [0, x]
> instead of array indices, using my closed-interval template. `left` only
> moves right when mid squared is less than x, so when the loop exits,
> `left` is the first integer whose square exceeds x — and `right`, which is
> left minus one, is the last integer whose square is at most x. That's
> exactly floor of sqrt(x), so I return `right`. One edge case: `mid * mid`
> overflows int for large x, so I cast to long, or compare `mid > x / mid`
> instead. O(log x) time, O(1) space."
