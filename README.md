# Algo Notes
 
Personal LeetCode mistake notebook (Java). The goal isn't to collect answers,
but to track how my understanding evolves across multiple passes over the same problem.
 
## Structure
 
Grouped by data structure / problem model. One markdown file per problem.
 
```
algo_notes/
├── binary_search/
│   ├── LC704.md
│   └── assets/704.png
├── tree/
├── dp/
└── ...
```
 
## How each note works
 
- **一刷 / 二刷 / 三刷** — each pass adds a section to the *same* file (never a new file or branch).
- Mistakes are marked **inline as `// ❌` comments** next to the buggy lines; ❌ / ✅ shows how each pass went.
- Correct solution + takeaways sit at the **bottom**, so I recall before I peek.
